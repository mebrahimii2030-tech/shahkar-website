# راهنمای راه‌اندازی پنل مشتریان روی Cloudflare Workers

با بررسی دقیق دیدیم پروژه‌ی `shahkar-website` روی داشبورد Cloudflare از نوع **Worker** است (نه Pages) و به ریپازیتوری گیت‌هاب `mebrahimii2030-tech/shahkar-website` وصل است. این راهنما مخصوص همین حالت است.

## چیزهایی که باید حذف/جایگزین کنی

- پوشه‌ی قدیمی `functions/` (اگر قبلاً اضافه کرده بودی) را کامل **حذف کن** — آن ساختار فقط برای Cloudflare Pages کار می‌کند، نه Workers، و اگر بماند هم استفاده نمی‌شود و هم سورس‌کدش به‌صورت عمومی قابل دانلود می‌شود.
- اگر قبلاً یک `wrangler.jsonc` دیگری (مثلاً با فیلد `pages_build_output_dir`) ساخته بودی، با فایل جدید `wrangler.jsonc` که اینجا هست جایگزینش کن.

## فایل‌هایی که باید در ریشه پروژه قرار بگیرند

- `wrangler.jsonc` — تنظیمات اصلی Worker (نام پروژه، اتصال به D1، مسیر فایل استاتیک)
- `.assetsignore` — جلوگیری از عمومی‌شدن فایل‌های سورس (`src/`, `schema.sql`, همین راهنما و ...)
- `src/worker.js` — کل منطق سرور (API مشتریان/خودروها/مراجعات/قطعات + محافظت پنل مدیریت)، جایگزین پوشه قدیمی `functions/`
- بقیه فایل‌ها (`schema.sql`, `panel-admin.html`, `panel-customer.html`, `customer.html`, `css/panel.css`, `js/panel-*.js`, `js/combobox.js`, `js/jalali.js`, `js/customer-view.js`, `js/panel-data.js`) دقیقاً مثل قبل، کنار فایل‌های فعلی سایتت.

## مرحله ۱: ساخت پایگاه‌داده D1 (اگر هنوز نساختی)

از قبل با چت‌جی‌پی‌تی این کار انجام شده: دیتابیس `shamkar-db` با شناسه‌ی
`b3e72150-6858-4a63-ba8d-957a8808af01` ساخته شده و جدول‌ها هم داخلش هستند.
اگر جدول‌ها را نساختی، محتوای `schema.sql` را در تب **Console** همان دیتابیس (روی Cloudflare Dashboard → Workers & Pages → D1 → shamkar-db) اجرا کن.

اگر شناسه دیتابیس‌ات چیز دیگری بود (نه همین که بالا نوشته شد)، توی `wrangler.jsonc` مقدار `database_id` را با شناسه‌ی درستت جایگزین کن — از دستور `npx wrangler d1 list` یا از داشبورد D1 می‌تونی ببینیش.

## مرحله ۲: تنظیم رمز عبور پنل مدیریت

این را حتماً از **داشبورد** انجام بده، نه از فایل کد (چون رمز نباید داخل گیت‌هاب و به‌صورت عمومی ذخیره بشه):

۱. برو به پروژه `shahkar-website` در Cloudflare
۲. **Settings → Variables and Secrets**
۳. یک متغیر جدید بساز: نام `ADMIN_PASSWORD`، مقدار یک رمز قوی، نوعش را **Secret** بگذار
۴. ذخیره کن

این مقدار مستقل از گیت‌هاب است و با هر بار `push` از بین نمی‌رود.

## مرحله ۳: انتشار

فایل‌ها را به ریپازیتوری `mebrahimii2030-tech/shahkar-website` اضافه کن و:

```
git add .
git commit -m "افزودن پنل مشتریان (سازگار با Cloudflare Workers)"
git push
```

چون پروژه به گیت‌هاب وصل است، Cloudflare خودش build و deploy می‌کند (همان چیزی که در تصویر داشبورد به‌عنوان build minutes دیده می‌شود).

## اگر دستی هم خواستی تست کنی

```
npx wrangler deploy
```

این دستور را از داخل پوشه‌ی پروژه (همان‌جایی که `wrangler.jsonc` هست) اجرا کن. اگر پیام خطا داد، متن کامل خطا را بفرست تا دقیق بررسی کنیم.

## آدرس‌های پنل بعد از انتشار

- **پنل مدیریت (فقط خودت، با رمز عبور):** `https://shahkar-website.m-ebrahimii2030.workers.dev/panel-admin.html` (یا دامنه اختصاصی‌ات اگر وصل کردی)
- **صفحه اختصاصی هر مشتری (بدون رمز، فقط با لینک):** `.../customer.html?code=XXXXXXXXXX`

## نکته درباره تاریخ‌ها

همه تاریخ‌ها (تاریخ مراجعه، موعد تعویض قطعه) به‌صورت **شمسی** وارد و نمایش داده می‌شوند (مثلاً `۱۴۰۵/۰۴/۲۸`)؛ تبدیل به میلادی فقط پشت صحنه برای ذخیره‌سازی انجام می‌شود.

## نکته درباره Encoding فارسی

اگر فایل‌ها را با ادیتوری مثل Notepad ساده در ویندوز باز/ذخیره کردی و کامنت‌های فارسی به‌هم‌ریخته دیدی، هنگام ذخیره باید Encoding را روی **UTF-8** بگذاری (نه ANSI). در VS Code این تنظیم پایین صفحه (گوشه راست پایین) قابل تغییر است. فایل‌هایی که من اینجا تحویل دادم همه از ابتدا UTF-8 هستند؛ این نکته فقط برای زمانی‌ست که خودت فایلی را در ویندوز ویرایش و دوباره ذخیره می‌کنی.
